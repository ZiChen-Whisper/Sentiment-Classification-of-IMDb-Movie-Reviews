## 1. 任务背景

### 1.1 我们在做什么？

**任务名称**：IMDB电影评论情感分类

**一句话解释**：给计算机一段英文电影评论，让它自动判断这条评论是"好评"还是"差评"。

### 1.2 数据来源

数据来自 IMDB（互联网电影数据库），包含5万条用户评论：

| 数据集 | 数量 | 用途 |
|--------|------|------|
| 训练集 | 40,000条 | 用来"教"模型学会判断 |
| 验证集 | 5,000条 | 调参，检查是否学偏了 |
| 测试集 | 5,000条 | 最终考试，看模型真实水平 |

每条数据有两个字段：
- `text`：评论文本（英文）
- `label`：标签，0=差评，1=好评

### 1.3 为什么是二分类？

因为每个评论只有两种可能：好评（正向）或差评（负向）。这种"非黑即白"的任务叫做**二分类**（Binary Classification），是机器学习中最基础的任务之一。

## 2. 前置概念

### 2.1 特征提取（Feature Extraction）

计算机不认文字，只认数字。特征提取就是把文字变成数字的过程。

**举例**：
```
评论："This movie is great"
```

变成数字向量：
```
{"this": 1, "movie": 1, "is": 1, "great": 1} → [1, 1, 1, 1]
```

### 2.2 词频（Count）

数每个词在评论中出现了几次。

**举例**：
```
评论1："great movie great acting"
词表：{great, movie, acting}
词频向量：[2, 1, 1]  （great出现2次）
```

### 2.3 TF-IDF（词频-逆文档频率）

词频的问题：像"the"、"is"这种常见词在每个评论中都有高词频，但对判断情感没用。

TF-IDF解决这个问题的方法是：**一个词如果在当前评论中频繁出现，但在其他评论中很少出现，那这个词就很有区分力**。

- **TF（词频）**：这个词在这条评论中出现了几次
- **IDF（逆文档频率）**：log(总评论数 / 包含该词的评论数)

**举例**：
- "the"几乎每条评论都有，IDF低，TF-IDF得分低
- "masterpiece"只在好评中出现，IDF高，TF-IDF得分高

### 2.4 逻辑回归的sigmoid函数

逻辑回归输出的不是直接的0或1，而是一个概率值（0~1之间），然后通过一个阈值（通常是0.5）来决定是0还是1。

```
sigmoid(x) = 1 / (1 + e^(-x))
```

这个函数的特点是：把任意实数映射到(0, 1)区间，输出可以理解为"是好评的概率"。

### 2.5 朴素贝叶斯定理

贝叶斯定理：P(A|B) = P(B|A) × P(A) / P(B)

在情感分类中：
- P(好评|词) = P(词|好评) × P(好评) / P(词)
- 我们分别计算P(好评|词)和P(差评|词)，取大的那个

"朴素"的意思是：假设每个词的出现是独立的。这个假设虽然不符合现实（"not"和"good"显然有关联），但实践中效果还不错。

### 2.6 卡方检验（Chi-square Test）

用于衡量**一个特征和类别之间是否有显著的统计关联**。

- 卡方值大：这个词和情感标签强相关（比如"great"几乎只出现在好评中）
- 卡方值小：这个词和情感标签无关（比如"the"在好评差评中都出现）

### 2.7 评测指标

| 指标 | 含义 | 公式 |
|------|------|------|
| Accuracy（准确率） | 预测对的占总数的比例 | (正确预测数) / (总样本数) |
| Precision（精确率） | 预测为好评的评论中，真正是好评的比例 | TP / (TP + FP) |
| Recall（召回率） | 所有真正的好评中，被正确预测出来的比例 | TP / (TP + FN) |
| F1-Score | 精确率和召回率的调和平均，综合指标 | 2 × (P × R) / (P + R) |

- **TP（True Positive）**：真好评，预测为好评
- **TN（True Negative）**：真差评，预测为差评
- **FP（False Positive）**：真差评，但预测为好评（假阳性）
- **FN（False Negative）**：真好评，但预测为差评（假阴性）

## 4. 模型逐一讲解

### 4.1 模型1：产生式系统（SentiWordNet）

#### 4.1.1 这是什么？

这是**唯一一个不需要训练数据的模型**。它完全依赖"查词典"来判断情感，就像你遇到不认识的词查字典一样。

#### 4.1.2 SentiWordNet是什么？

SentiWordNet是一个给每个英语单词打上"情感分"的词典。每个词都有两个分数：
- `pos_score`：正向得分（0~1，越大越正面）
- `neg_score`：负向得分（0~1，越大越负面）

**举例**：
| 词 | pos_score | neg_score | 含义 |
|-----|-----------|-----------|------|
| great | 0.75 | 0.0 | 很正面 |
| terrible | 0.0 | 0.88 | 很负面 |
| movie | 0.0 | 0.0 | 中性词 |

#### 4.1.3 工作原理

整个流程可以用以下伪代码表示：

```
1. 对评论进行词性标注（每个词是名词、动词、还是形容词？）
2. 将词性标签转换为WordNet格式
3. 对每个词，在SentiWordNet中查找对应的情感得分
4. 累加所有词的正向得分和负向得分
5. 如果正向总分 ≥ 负向总分 → 预测为好评(1)，否则为差评(0)
```

**具体例子**：
```
评论："This movie is great but the acting is terrible"

词性标注结果：
- great (形容词): pos=0.75, neg=0.0
- terrible (形容词): pos=0.0, neg=0.88
- 其他词: 得分接近0

正向总分 = 0.75
负向总分 = 0.88
0.75 < 0.88 → 预测为差评
```

#### 4.1.4 代码实现核心

```python
def sentiwordnet_predict(text):
    # 分词和词性标注
    tokens = word_tokenize(text)
    tagged = nltk.pos_tag(tokens)
    
    pos_score = 0.0
    neg_score = 0.0
    
    for word, tag in tagged:
        # 转换为WordNet词性
        wn_tag = penn_to_wn(tag)
        if wn_tag is None:
            continue
        
        # 查找同义词集
        synsets = wn.synsets(word, pos=wn_tag)
        if not synsets:
            continue
        
        # 取第一个同义词集，获取情感得分
        synset = synsets[0]
        swn_synset = swn.senti_synset(synset.name())
        pos_score += swn_synset.pos_score()
        neg_score += swn_synset.neg_score()
    
    # 判断
    prediction = 1 if pos_score >= neg_score else 0
    return prediction, pos_score, neg_score
```

#### 4.1.5 性能

| 指标 | 值 |
|------|-----|
| 准确率 | 61.92% |
| Macro F1 | 59.00% |

#### 4.1.6 为什么准确率这么低？

| 问题 | 举例 | 说明 |
|------|------|------|
| **忽略上下文** | "not bad" | "bad"被计为负向，但实际表达正面 |
| **忽略语序** | "虽然演技差，但是剧情好" | 简单累加无法处理转折关系 |
| **同义词集选择问题** | "bank"可以是银行或河岸 | 取第一个同义词集可能不对 |
| **无法处理反讽** | "Oh great, another terrible sequel" | "great"在这里是反讽 |

### 4.2 模型2：朴素贝叶斯（词频特征）

#### 4.2.1 这是什么？

这是第一个**真正从数据中学习的模型**。它通过贝叶斯定理，根据训练数据中学习到的"每个词在好评/差评中出现的概率"来做判断。

#### 4.2.2 贝叶斯定理在情感分类中的应用

核心公式：
```
P(好评|词1,词2,...,词n) = P(好评) × P(词1|好评) × P(词2|好评) × ... × P(词n|好评)
```

**通俗理解**：
- 模型从训练数据中学到："great"这个词在好评中出现的概率是多少，在差评中出现的概率是多少
- 当新评论到来时，它计算："看到这些词出现的情况下，这条评论是好评的概率" vs "是差评的概率"
- 哪个概率大，就预测哪个

#### 4.2.3 工作流程

```
1. 用CountVectorizer将文本转为词频向量（最多50000个特征）
   → 每条评论变成一个长度为50000的向量
   
2. 训练MultinomialNB（多项式朴素贝叶斯）分类器
   → 模型学习每个词在好评/差评中的出现概率
   
3. 预测新评论
   → 计算P(好评|词)和P(差评|词)，取大的
```

#### 4.2.4 代码实现

```python
# 步骤1：词频特征提取
count_vec = CountVectorizer(max_features=50000)
X_train_count = count_vec.fit_transform(X_train_text)  # 训练集特征
X_test_count = count_vec.transform(X_test_text)        # 测试集特征

# 步骤2：训练朴素贝叶斯
nb_model = MultinomialNB()
nb_model.fit(X_train_count, y_train)

# 步骤3：预测
test_preds_nb = nb_model.predict(X_test_count)
```

#### 4.2.5 性能

| 指标 | 值 | 相比模型1 |
|------|-----|-----------|
| 准确率 | 86.42% | +24.5% |
| Macro F1 | 86.42% | +27.4% |

#### 4.2.6 为什么比模型1好这么多？

| 对比项 | 模型1（SentiWordNet） | 模型2（朴素贝叶斯） |
|--------|----------------------|-------------------|
| **知识来源** | 人工编写的词典 | 从4万条数据中自动学习 |
| **上下文** | 不考虑词的组合 | 通过词的组合概率间接考虑 |
| **领域适应** | 通用词典，不针对电影 | 从IMDB数据学习，更适应电影评论 |

**关键发现**：从数据中自动学习规律，远比人工设计规则有效。

### 4.3 模型3：逻辑回归（词频特征）

#### 4.3.1 这是什么？

逻辑回归是一个**线性分类器**。它的核心思想是：给每个特征（词）学一个权重，加权求和后通过sigmoid函数输出"好评"的概率。

#### 4.3.2 工作原理

数学公式：
```
P(好评|文本) = sigmoid(w₁×词1频次 + w₂×词2频次 + ... + b)
```

- `w₁, w₂, ...`：每个词的权重（模型通过学习得到）
- `b`：偏置项
- `sigmoid(x) = 1 / (1 + e^(-x))`：将任意实数映射到(0, 1)

**权重的含义**：
- 如果"great"的权重是正的且很大 → 出现"great"倾向于好评
- 如果"terrible"的权重是负的且绝对值很大 → 出现"terrible"倾向于差评
- 如果"movie"的权重接近0 → "movie"对判断没什么帮助

#### 4.3.3 与朴素贝叶斯的区别

| 对比项 | 朴素贝叶斯 | 逻辑回归 |
|--------|-----------|---------|
| **建模方式** | 生成式：建模P(词|类别) | 判别式：直接建模P(类别|词) |
| **假设** | 假设词之间独立 | 不假设独立性 |
| **一般表现** | 数据量少时可能更好 | 数据量大时通常更好 |

**通俗理解**：
- 朴素贝叶斯像"从结果反推原因"（好评通常包含哪些词？）
- 逻辑回归像"从原因直接判断结果"（看到这些词，是好评的概率是多少？）

#### 4.3.4 代码实现

```python
# 使用和模型2相同的词频特征
lr_count = LogisticRegression(max_iter=1000, random_state=42)
lr_count.fit(X_train_count, y_train)  # 训练

test_preds_lr_count = lr_count.predict(X_test_count)  # 预测
```

|------|-----|-----------|
| 准确率 | 88.90% | +2.5% |
| Macro F1 | 88.90% | +2.5% |

### 4.4 模型4：逻辑回归（TF-IDF特征）

#### 4.4.1 这是什么？

和模型3使用相同的逻辑回归算法，但**特征从词频换成了TF-IDF**。

#### 4.4.2 为什么要用TF-IDF？

**词频的问题**：
- "the"、"is"等常见词在每个文档中都有高词频
- 但它们对情感判断毫无帮助
- 它们会"淹没"真正有区分力的词

**TF-IDF的解决方式**：
```
TF-IDF(词, 文档) = TF(词, 文档) × IDF(词)
IDF(词) = log(总文档数 / 包含该词的文档数)
```

- 常见词（如"the"）→ 出现在很多文档中 → IDF低 → TF-IDF低
- 罕见词（如"masterpiece"）→ 只出现在少数文档中 → IDF高 → TF-IDF高

**举例**：
| 词 | 词频 | TF-IDF | 说明 |
|-----|------|--------|------|
| the | 5.0 | 0.2 | 常见词，被大幅降低权重 |
| great | 2.0 | 1.5 | 情感词，保持较高权重 |
| masterpiece | 1.0 | 2.8 | 区分性强，权重反而更高 |

#### 4.4.3 代码实现

```python
# 步骤1：TF-IDF特征提取
tfidf_vec = TfidfVectorizer(max_features=50000)
X_train_tfidf = tfidf_vec.fit_transform(X_train_text)
X_test_tfidf = tfidf_vec.transform(X_test_text)

# 步骤2：训练逻辑回归（和模型3相同）
lr_tfidf = LogisticRegression(max_iter=1000, random_state=42)
lr_tfidf.fit(X_train_tfidf, y_train)

# 步骤3：预测
test_preds_lr_tfidf = lr_tfidf.predict(X_test_tfidf)
```

#### 4.4.4 性能

| 指标 | 值 | 相比模型3 |
|------|-----|-----------|
| 准确率 | 89.54% | +0.64% |
| Macro F1 | 89.54% | +0.64% |

#### 4.4.5 提升不大但为什么还用TF-IDF？

- TF-IDF是**更合理的特征表示方式**
- 在数据量更大或任务更复杂时，优势会更明显
- 它是NLP任务中的标准做法

**结论**：TF-IDF是传统方法中最好的特征表示方式（准确率89.54%，是所有传统模型中最高的）。

### 4.5 模型5：逻辑回归 + 特征选择（K=200）

#### 4.5.1 这是什么？

在50000个TF-IDF特征中，很多是噪声。这个模型用**卡方检验**筛选出最好的200个特征。

#### 4.5.2 为什么要做特征选择？

**问题**：50000个特征中，只有少部分是真正有用的
- 大部分词对情感判断没有帮助
- 噪声特征会干扰模型学习
- 特征越多，计算成本越高

**解决方案**：只保留最好的K个特征

#### 4.5.3 卡方检验（Chi-square Test）

卡方检验衡量的是：**某个特征与类别之间是否有显著的统计关联**。

**计算逻辑**：
```
对于每个词，计算它在好评和差评中的实际出现次数 vs 期望出现次数
差异越大 → 卡方值越大 → 这个词越有区分力
```

**举例**：
| 词 | 好评中出现次数 | 差评中出现次数 | 卡方值 | 说明 |
|-----|---------------|---------------|--------|------|
| great | 1500 | 200 | 很大 | 强区分力 |
| terrible | 100 | 1200 | 很大 | 强区分力 |
| the | 18000 | 17500 | 很小 | 无区分力 |

#### 4.5.4 SelectKBest工具

`SelectKBest(chi2, k=200)` = "用卡方检验选择最好的200个特征"

#### 4.5.5 代码实现

```python
# 步骤1：特征选择（从50000降到200）
selector_200 = SelectKBest(chi2, k=200)
X_train_sel200 = selector_200.fit_transform(X_train_tfidf, y_train)
X_test_sel200 = selector_200.transform(X_test_tfidf)

# 步骤2：训练逻辑回归
lr_sel200 = LogisticRegression(max_iter=1000, random_state=42)
lr_sel200.fit(X_train_sel200, y_train)

# 步骤3：预测
test_preds_sel200 = lr_sel200.predict(X_test_sel200)
```

#### 4.5.6 性能

| 指标 | 值 | 相比模型4 | 特征维度 |
|------|-----|-----------|---------|
| 准确率 | 84.34% | -5.2% | 200（降低99.6%） |
| Macro F1 | 84.33% | -5.2% | |

#### 4.5.7 选中的特征举例

选中的200个特征包括：amazing, annoying, atrocious, avoid, abysmal 等，都是具有明确情感倾向的词。

**发现**：K=200时，特征维度降低了99.6%，但性能只下降5.2个百分点。这说明大部分情感信息集中在少量关键特征中，但200个特征还不够。

### 4.6 模型6：逻辑回归 + 特征选择（K=2000）

#### 4.6.1 这是什么？

和模型5相同的方法，但保留**最好的2000个特征**而不是200个。

#### 4.6.2 为什么选择K=2000？

通过对比不同K值的实验结果：

| K值 | 特征维度 | 准确率 | 相对全特征的性能保留 | 特征降低比例 |
|-----|---------|--------|-------------------|-------------|
| 50000 | 50000 | 89.54% | 100% | 0% |
| 2000 | 2000 | 88.68% | 98.8% | 96% |
| 200 | 200 | 84.34% | 94.0% | 99.6% |

**关键发现**：K=2000是一个极好的平衡点：
- 只用4%的特征（2000/50000），就保留了98.8%的性能
- 特征维度降低96%，大幅降低计算成本
- 准确率仅比全特征低0.86个百分点

### 4.7 模型7：逻辑回归 + 新特征设计

#### 4.7.1 这是什么？

除了TF-IDF这些"文本向量"特征，我们还**手工设计了8个直觉上有用的特征**，和TF-IDF拼接在一起训练模型。

#### 4.7.2 设计了哪8个新特征？

| 特征 | 含义 | 直观理解 | 逻辑回归系数 | 解读 |
|------|------|---------|------------|------|
| word_count | 评论词数 | 长评论可能更认真 | +0.2145 | 长评论略偏向好评 |
| avg_word_len | 平均词长 | 用词复杂程度 | -0.0528 | 用短词多的略偏差评 |
| exclamation_count | 感叹号数量 | 情感强烈程度 | +0.0292 | 感叹号略偏好评 |
| upper_ratio | 大写字母比例 | 愤怒程度（如"SO BAD"） | -0.0283 | 大写多略偏差评 |
| negation_count | 否定词数量 | 否定表达 | **-0.2467** | 否定词多强烈偏向差评 |
| sentiment_ratio | 情感词比例 | 评论中有多少情感词 | +0.0157 | 影响很小 |
| pos_sentiment_ratio | 正向情感词比例 | 正面情感词占比 | +0.1658 | 正向词多偏好评 |
| neg_sentiment_ratio | 负向情感词比例 | 负面情感词占比 | **-0.2602** | 负向词多强烈偏向差评 |

#### 4.7.5 为什么新特征没有显著提升？

**原因**：维度不平衡
- TF-IDF有50000维
- 新特征只有8维

### 4.8 模型8：大语言模型（LLM）

#### 4.8.1 这是什么？

这是最"现代"的方法，直接使用预训练的大语言模型（如DeepSeek）来进行情感分类，**无需任何训练数据**。

#### 4.8.2 模型8a：Few-shot Prompt策略

**什么是Few-shot？**

给大语言模型几个"示例"，让它学会任务格式，然后对新输入进行判断。就像考试前给几道例题，让学生理解题型。

**Prompt设计示例**：
```
You are a sentiment analysis expert. Classify the following movie review as either "positive" or "negative".

Here are some examples:

Review: This movie was absolutely wonderful! The acting was superb.
Answer: positive

Review: Terrible film. The plot made no sense.
Answer: negative

... (共6个示例)

Now classify the following review:
Review: {待分类的评论}
Answer:
```

**性能**：
- 准确率：95.38%
- 远超所有传统方法！

**为什么这么好？**
1. 大语言模型通过预训练学习了海量语言知识
2. 6个示例足以让它理解任务
3. 能理解否定（"not bad"）、反讽等复杂语义
4. 这是传统方法做不到的

#### 4.8.3 模型8b：结构化输出（框架表示法）

**什么是框架表示法？**

不仅输出"好评/差评"，还输出更丰富的结构化信息：

```json
{
  "topic": ["acting", "plot"],
  "sentiment": "positive",
  "sentiment_score": 0.85,
  "key_phrases": ["superb acting", "captivating story"],
  "aspects": [
    {"aspect": "acting", "sentiment": "positive", "evidence": "The acting was superb"},
    {"aspect": "plot", "sentiment": "positive", "evidence": "captivating from start to finish"}
  ]
}
```

**为什么这很有意义？**

一条评论可能说"演技很好但剧情太差"。传统方法只能给一个整体判断，但框架表示可以**对不同方面给出不同情感**，这是更细粒度的分析。

**性能**：
- 准确率：90%（20条数据）
- 高置信度(≥0.7)的预测准确率94.74%
- 低置信度的预测更容易出错——模型"知道自己不确定"

#### 4.8.4 代码实现（Few-shot）

```python
# 注意：需要API Key

Answer: negative

... (6个示例)

Review: {review_text}
Answer:
```
