## 二、前置知识：你需要理解的核心概念

### 2.1 什么是"情感分类"？

想象你是一个电影网站的运营，每天有上万条用户评论，你不可能一条条看。你希望有一个程序，能自动把评论分成"好评"和"差评"两类，这就是情感分类。

- **正向（positive）**：如 "This movie is amazing! I loved every minute of it."
- **负向（negative）**：如 "Terrible film. Complete waste of time."

### 2.2 什么是"特征"？

计算机不认识文字，它只懂数字。所以我们需要把文本"翻译"成数字，这个翻译过程就叫**特征提取**。

比如一条评论 "This movie is great"，我们可以：
- **词频特征（Count）**：统计每个词出现的次数 → {"this":1, "movie":1, "is":1, "great":1}
- **TF-IDF特征**：在词频的基础上，降低常见词（如"the"、"is"）的权重，提高区分性词（如"great"）的权重

### 2.3 什么是"词频"（Count）？

就是数每个词出现了几次。假设有两条评论：
- 好评："great movie great acting"
- 差评："terrible movie boring acting"

建立一个词表：{great, movie, acting, terrible, boring}

好评的词频向量：[2, 1, 1, 0, 0]  （great出现2次，movie 1次，acting 1次，terrible和boring没出现）
差评的词频向量：[0, 1, 1, 1, 1]

这样，文本就变成了数字向量，计算机就能处理了。

### 2.4 什么是"TF-IDF"？

TF-IDF = Term Frequency - Inverse Document Frequency（词频-逆文档频率）

核心思想：**一个词如果在一篇文档中出现很多次，但在其他文档中很少出现，那这个词就很有区分性，应该给更高的权重。**

- **TF（词频）**：这个词在这篇文档中出现了几次
- **IDF（逆文档频率）**：log(总文档数 / 包含该词的文档数)。一个词越常见（如"the"），IDF越低；越罕见（如"masterpiece"），IDF越高
- **TF-IDF = TF × IDF**

举例："the"在每个文档中都出现，IDF很低，TF-IDF接近0；"masterpiece"只在好评中出现，IDF高，TF-IDF也高。这样就能自动突出有区分力的词。

### 2.5 什么是"PMI"（点互信息）？

PMI衡量的是**一个词和某个类别之间的关联程度**。

公式：PMI(word, class) = log(P(word和class同时出现) / (P(word) × P(class)))

- PMI > 0：这个词和这个类别正相关（比随机出现更频繁）
- PMI ≈ 0：没有关联
- PMI < 0：负相关

举例：如果"masterpiece"几乎只出现在好评中，那 PMI(masterpiece, 正向) 就会很高；如果"terrible"几乎只出现在差评中，那 PMI(terrible, 负向) 就会很高。

### 2.6 什么是"词形还原"（Lemmatization）？

把一个词的不同形式还原成它的"原形"（词元）：
- watched → watch
- better → good
- cats → cat

这样做的好处：减少特征数量，把"watch"、"watched"、"watching"都当作同一个词处理。

### 2.7 什么是"停用词"（Stop Words）？

那些出现频率极高但没什么实际意义的词，如 "the", "is", "at", "which", "on"。去掉它们可以减少噪声，降低特征维度。但要注意保留否定词（如 "not", "no"），因为它们对情感判断很重要——"not bad"和"bad"完全不同！

### 3.3 PMI分析——更有趣的发现

**什么是PMI**：
PMI（Pointwise Mutual Information，点互信息）衡量的是**一个词和某个类别之间的关联程度**。

公式：`PMI(word, class) = log(P(word和class同时出现) / (P(word) × P(class)))`
- PMI > 0：这个词和这个类别正相关（比随机出现更频繁）
- PMI ≈ 0：没有关联
- PMI < 0：负相关

**我们是怎么做的**：

**第1步：计算每个词在正/负向文档中出现的文档数**
```python
# 注意：这里是"文档频率"，不是"词频"
# 即一个词在多少条评论中出现过，而不是出现多少次
pos_doc_freq = Counter()  # 词在正向文档中出现的文档数
neg_doc_freq = Counter()  # 词在负向文档中出现的文档数

for tokens in pos_tokens_list:  # 每条评论的token列表（不去停用词）
    unique_tokens = set(tokens)  # 每条评论中每个词只计一次
    for t in unique_tokens:
        pos_doc_freq[t] += 1

for tokens in neg_tokens_list:
    unique_tokens = set(tokens)
    for t in unique_tokens:
        neg_doc_freq[t] += 1
```

**第2步：计算PMI**
```python
def compute_pmi(word, doc_freq_class, class_docs, N_docs):
    """
    word: 要计算的词
    doc_freq_class: 该词在目标类别文档中的出现次数
    class_docs: 目标类别的文档总数
    N_docs: 总文档数
    """
    # P(word, class) = 该词在目标类别文档中出现的文档数 / 总文档数
    p_word_class = doc_freq_class[word] / N_docs
    
    # P(word) = 该词在所有文档中出现的文档数 / 总文档数
    total_doc_freq = pos_doc_freq[word] + neg_doc_freq[word]
    p_word = total_doc_freq / N_docs
    
    # P(class) = 目标类别文档数 / 总文档数
    p_class = class_docs / N_docs
    
    if p_word_class == 0 or p_word == 0:
        return -float('inf')
    
    return math.log(p_word_class / (p_word * p_class))
```

**为什么用文档频率而不是词频？**
因为如果一条评论中"great"出现了10次，不代表它比出现1次的评论更有信息量。文档频率更能反映"这个词是否与某类评论相关"。

**第3步：过滤低频噪声词，计算PMI排名**
```python
# 只考虑出现次数>=50的词（过滤低频噪声）

**这说明**：PMI比简单词频更能捕捉情感关联，专有名词（电影名、人名）具有强烈的情感指向性。

## 四、阶段二：数据预处理——我们做了什么？怎么做的？

### 4.1 预处理流程

**为什么要预处理？**
原始评论中有很多"噪声"，如果直接输入模型，会让模型学习到错误的规律。比如：
- `<br />` HTML标签对情感判断没有任何帮助
- "Movie"和"movie"应该被当作同一个词
- "the"、"is"等停用词在每个文档中都出现，但它们没有区分力

**我们的预处理流水线**：

```
原始文本 → 去HTML标签 → 去标点符号 → 转小写 → 去停用词 → 词形还原 → 干净文本
```

**具体步骤**：

**第1步：文本清洗（去HTML、去标点、转小写）**
```python
from bs4 import BeautifulSoup
import re

def clean_text(text):
    # 1. 去除HTML标签（如<br />）
    # 使用BeautifulSoup库，它会自动解析HTML并提取纯文本
    text = BeautifulSoup(text, "lxml").text
    
    # 2. 去除标点符号和特殊字符
    # [^\w\s] 表示"非字母数字且非空白字符"，即标点符号
    text = re.sub(r'[^\w\s]', '', text)
    
    # 3. 转小写
    text = text.lower()
    
    # 4. 合并多余空格
    text = re.sub(r'\s+', ' ', text).strip()
    return text
```

**第2步：停用词处理**
```python
from nltk.corpus import stopwords

stopword = set(stopwords.words('english'))

def remove_stopwords(text):
    tokens = text.split()  # 按空格分词
    # 去除停用词
    tokens = [t for t in tokens if t not in stopword]
    return tokens
```

**注意**：我们保留了否定词（如"not"、"no"）吗？
NLTK默认停用词表**包含**否定词。但在情感分析中，否定词非常重要——"not bad"和"bad"完全不同！
不过，我们的代码中直接使用了默认停用词表，这意味着否定词也被去除了。这是可以通过自定义停用词表来优化的。

**第3步：词形还原（Lemmatization）**
```python
from nltk.stem import WordNetLemmatizer

lemmatizer = WordNetLemmatizer()

def lemmatize_tokens(tokens):
    return [lemmatizer.lemmatize(t) for t in tokens]
```

词形还原把一个词的不同形式还原成"原形"：
- watched → watch
- cats → cat
- better → better（注意：WordNetLemmatizer默认把词当作名词处理，如果要正确还原形容词，需要提供词性参数）

**第4步：完整预处理流水线**
```python
def preprocess_pipeline(text):
    """完整预处理流程：清洗 -> 去停用词 -> 词形还原"""
    cleaned = clean_text(text)           # 清洗
    tokens = remove_stopwords(cleaned)   # 去停用词
    lemmatized = lemmatize_tokens(tokens) # 词形还原
    return ' '.join(lemmatized)          # 重新拼接成字符串
```

**第5步：批量处理数据集**
```python
from tqdm import tqdm

def preprocess_dataframe(df, name):
    """对整个DataFrame进行预处理"""
    print(f"  预处理 {name} ({len(df)} 条)...")
    cleaned_texts = []
    raw_cleaned_texts = []

    for text in tqdm(df['text'], desc=f"  {name}"):
        # 完整预处理版本（清洗+去停用词+词形还原）
        full_processed = preprocess_pipeline(text)
        cleaned_texts.append(full_processed)

        # 仅清洗版本（保留停用词，供某些模型使用）
        raw_cleaned = clean_text(text)
        raw_cleaned_texts.append(raw_cleaned)

    df_processed = df.copy()
    df_processed['text_cleaned'] = cleaned_texts        # 完整预处理
    df_processed['text_cleaned_raw'] = raw_cleaned_texts # 仅清洗
    return df_processed

# 处理三个数据集
train_processed = preprocess_dataframe(train_df, '训练集')
valid_processed = preprocess_dataframe(valid_df, '验证集')
test_processed = preprocess_dataframe(test_df, '测试集')
```

**为什么要保存两个版本？**
- `text_cleaned`：适用于需要精简特征的模型（朴素贝叶斯、逻辑回归等）
- `text_cleaned_raw`：保留了停用词，适用于需要完整语义信息的模型（如SentiWordNet需要"not"来理解否定结构）

**第6步：保存预处理结果**
```python

valid_processed.to_csv('data/valid_processed.csv', index=False)

### 模型1：基于规则的产生式系统（SentiWordNet）

#### 核心思想

不训练任何模型，完全靠"查词典"来判断情感。

#### 什么是SentiWordNet？

SentiWordNet是一个给每个英语单词打"情感分"的词典。每个词都有：
- 正向得分（pos_score）：0~1，越大越正面
- 负向得分（neg_score）：0~1，越大越负面

比如：
- "great"：pos_score=0.75, neg_score=0.0 → 很正面
- "terrible"：pos_score=0.0, neg_score=0.88 → 很负面
- "movie"：pos_score=0.0, neg_score=0.0 → 中性词

#### 我们是怎么做的？

**第1步：定义词性标签映射函数**

SentiWordNet使用WordNet的词性（ADJ、NOUN、VERB、ADV），而nltk.pos_tag输出的是Penn Treebank词性标签（JJ、NN、VB等），需要映射：

```python
from nltk.corpus import wordnet as wn

def penn_to_wn(tag):
    """将Penn Treebank词性标签映射为WordNet词性"""
    if tag.startswith('J'):
        return wn.ADJ      # 形容词
    elif tag.startswith('N'):
        return wn.NOUN     # 名词
    elif tag.startswith('R'):
        return wn.ADV      # 副词
    elif tag.startswith('V'):
        return wn.VERB     # 动词
    return None
```

**第2步：定义情感预测函数**

```python
from nltk.corpus import sentiwordnet as swn
from nltk.tokenize import word_tokenize
import nltk

def sentiwordnet_predict(text):
    """使用SentiWordNet计算文本情感得分"""
    # 1. 分词
    tokens = word_tokenize(str(text))
    
    # 2. 词性标注
    tagged = nltk.pos_tag(tokens)
    
    pos_score = 0.0  # 正向总分
    neg_score = 0.0  # 负向总分
    pos_word_count = 0  # 正向词数
    neg_word_count = 0  # 负向词数
    
    # 3. 遍历每个词，查询情感得分
    for word, tag in tagged:
        wn_tag = penn_to_wn(tag)  # 映射词性
        if wn_tag is None:
            continue
        
        # 获取WordNet同义词集
        synsets = wn.synsets(word, pos=wn_tag)
        if not synsets:
            continue
        
        # 取第一个同义词集（可能有多个含义，我们取最常见的）
        synset = synsets[0]
        
        # 查询SentiWordNet得分
        swn_synset = swn.senti_synset(synset.name())
        p = swn_synset.pos_score()
        n = swn_synset.neg_score()
        
        pos_score += p
        neg_score += n
        
        if p > n:
            pos_word_count += 1
        elif n > p:
            neg_word_count += 1
    
    # 4. 根据总分判断情感
    prediction = 1 if pos_score >= neg_score else 0
    return prediction, pos_score, neg_score, pos_word_count, neg_word_count
```

**第3步：对验证集和测试集进行预测**

```python
from tqdm import tqdm

# 预测验证集
valid_preds_swn = []
valid_swn_scores = []
for text in tqdm(X_valid_raw, desc="验证集"):
    pred, pos_s, neg_s, pos_w, neg_w = sentiwordnet_predict(text)
    valid_preds_swn.append(pred)
    valid_swn_scores.append({'pos_score': pos_s, 'neg_score': neg_s, 
                             'pos_word_count': pos_w, 'neg_word_count': neg_w})

# 预测测试集（同样方式）
test_preds_swn = []
test_swn_scores = []
for text in tqdm(X_test_raw, desc="测试集"):
    pred, pos_s, neg_s, pos_w, neg_w = sentiwordnet_predict(text)
    test_preds_swn.append(pred)
    test_swn_scores.append({'pos_score': pos_s, 'neg_score': neg_s, 
                            'pos_word_count': pos_w, 'neg_word_count': neg_w})
```

**第4步：评估**

```python

print(classification_report(y_test, test_preds_swn, target_names=['负向', '正向']))
```

#### 举例

评论："This movie is great but the acting is terrible"

- great: pos=0.75, neg=0.0
- terrible: pos=0.0, neg=0.88
- 其他词（this, movie, is, but, the, acting）: 大多是0或很小

正向总分 ≈ 0.75，负向总分 ≈ 0.88 → 判断为**差评**

### 模型2：朴素贝叶斯（词频特征）

#### 核心思想

基于**贝叶斯定理**，计算"给定这些词出现的情况下，评论是好评/差评的概率"，选概率大的那个。

#### 什么是朴素贝叶斯？

朴素贝叶斯之所以"朴素"，是因为它做了一个很强的假设：**每个词的出现是独立的**。虽然这个假设在现实中不成立（"not"和"good"显然不独立），但在实践中效果还不错。

公式：P(好评|词1,词2,...,词n) ∝ P(好评) × P(词1|好评) × P(词2|好评) × ... × P(词n|好评)

#### 我们是怎么做的？

**第1步：文本向量化**

```python
from sklearn.feature_extraction.text import CountVectorizer

# CountVectorizer将文本转为词频向量
# max_features=50000 表示最多保留50000个最常见的词作为特征
count_vec = CountVectorizer(max_features=50000)

# fit_transform：在训练集上学习词表，并转换为词频矩阵
X_train_count = count_vec.fit_transform(X_train_text)

# transform：使用训练集学到的词表，将验证集/测试集转换为词频矩阵
X_valid_count = count_vec.transform(X_valid_text)
X_test_count = count_vec.transform(X_test_text)

print(f"词表大小: {len(count_vec.vocabulary_)}")
```

**词频向量是什么样的？**
假设有词表：{"great":0, "movie":1, "bad":2, "acting":3}
评论"great movie great acting"会被转换为：[2, 1, 0, 1]
- great出现2次 → 索引0的值为2
- movie出现1次 → 索引1的值为1
- bad出现0次 → 索引2的值为0
- acting出现1次 → 索引3的值为1

**第2步：训练朴素贝叶斯分类器**

```python
from sklearn.naive_bayes import MultinomialNB

# 创建多项式朴素贝叶斯模型
nb_model = MultinomialNB()

# 在训练集上训练
nb_model.fit(X_train_count, y_train)
```

**第3步：预测与评估**

```python

test_preds_nb = nb_model.predict(X_test_count)

# 评估
print(classification_report(y_test, test_preds_nb, target_names=['负向', '正向']))
```

#### 结果：准确率 86.42%

比规则方法提升了24.5个百分点！这说明**从数据中自动学习规律，远比人工设计规则有效**。

### 模型3：逻辑回归（词频特征）

#### 核心思想

逻辑回归是一个线性分类器：给每个特征（词）学一个权重，加权求和后通过sigmoid函数，输出"好评"的概率。

#### 什么是逻辑回归？

公式：P(好评|文本) = sigmoid(w₁×词1频次 + w₂×词2频次 + ... + b)

- 如果"great"的权重w是正的且很大，说明出现"great"倾向于好评
- 如果"terrible"的权重w是负的且绝对值很大，说明出现"terrible"倾向于差评

与朴素贝叶斯的区别：朴素贝叶斯是"生成式"模型（建模P(词|类别)），逻辑回归是"判别式"模型（直接建模P(类别|词)），通常判别式模型效果更好。

#### 我们是怎么做的？

**使用相同的词频特征，但换用逻辑回归模型**：

```python
from sklearn.linear_model import LogisticRegression

# 创建逻辑回归模型
# max_iter=1000：最大迭代次数，防止不收敛
# random_state=42：随机种子，保证结果可复现
lr_count = LogisticRegression(max_iter=1000, random_state=42)

# 训练
lr_count.fit(X_train_count, y_train)

# 预测
valid_preds_lr_count = lr_count.predict(X_valid_count)
test_preds_lr_count = lr_count.predict(X_test_count)

# 评估
print(classification_report(y_test, test_preds_lr_count, target_names=['负向', '正向']))
```

#### 结果：准确率 88.90%

比朴素贝叶斯提升2.5个百分点。逻辑回归通过学习特征权重，能更好地区分情感词与非情感词的贡献。

### 模型4：逻辑回归（TF-IDF特征）

#### 核心思想

和模型3一样用逻辑回归，但把词频特征换成TF-IDF特征。

#### 为什么要换？

词频特征的问题：像"the"、"is"这样的常见词在每个文档中都有很高的频次，但它们对情感判断毫无帮助，反而会"淹没"真正有区分力的词。

TF-IDF通过**降低常见词的权重、提升区分性词的权重**来解决这个问题。

TF = 词在当前文档中出现的次数
IDF = log(总文档数 / 包含该词的文档数)
TF-IDF = TF × IDF

#### 我们是怎么做的？

```python
from sklearn.feature_extraction.text import TfidfVectorizer

# TfidfVectorizer将文本转为TF-IDF向量
tfidf_vec = TfidfVectorizer(max_features=50000)

# 在训练集上学习词表并计算IDF
X_train_tfidf = tfidf_vec.fit_transform(X_train_text)

# 使用训练集的IDF值转换验证集/测试集
X_valid_tfidf = tfidf_vec.transform(X_valid_text)
X_test_tfidf = tfidf_vec.transform(X_test_text)

print(f"TF-IDF特征维度: {X_train_tfidf.shape[1]}")

# 训练逻辑回归
lr_tfidf = LogisticRegression(max_iter=1000, random_state=42)
lr_tfidf.fit(X_train_tfidf, y_train)

# 预测与评估
valid_preds_lr_tfidf = lr_tfidf.predict(X_valid_tfidf)
test_preds_lr_tfidf = lr_tfidf.predict(X_test_tfidf)
print(classification_report(y_test, test_preds_lr_tfidf, target_names=['负向', '正向']))
```

#### 结果：准确率 89.54%

比词频+逻辑回归提升0.64个百分点。虽然提升不大，但TF-IDF是更合理的特征表示方式。

### 模型5-6：逻辑回归 + 特征选择

在50000个TF-IDF特征中，可能很多是噪声。我们用**卡方检验（chi2）**来筛选出最具有区分力的特征。

### 模型7：逻辑回归 + 新特征设计

#### 核心思想

除了TF-IDF这些"文本向量"特征，我们还可以手工设计一些**直觉上有用的特征**，和TF-IDF拼接在一起。

#### 设计了哪8个新特征？

#### 我们是怎么做的？

**第1步：定义特征提取函数**

```python
def extract_new_features(df):
    """提取新特征"""
    features = []
    negation_words = {'not', 'no', 'never', 'neither', 'nobody', 'nothing',
                      'nowhere', 'nor', "n't", 'cannot', 'hardly', 'scarcely'}

    for _, row in tqdm(df.iterrows(), total=len(df), desc="提取特征"):
        text = str(row['text'])              # 原始文本
        cleaned = str(row['text_cleaned_raw'])  # 清洗后文本（保留停用词）
        tokens = cleaned.split()

        # 特征1：评论词数
        word_count = len(tokens)

        # 特征2：平均词长
        avg_word_len = np.mean([len(w) for w in tokens]) if tokens else 0

        # 特征3：感叹号数量（从原始文本）
        exclamation_count = text.count('!')

        # 特征4：大写字母比例（从原始文本）
        upper_count = sum(1 for c in text if c.isupper())
        upper_ratio = upper_count / len(text) if len(text) > 0 else 0

        # 特征5：否定词数量
        neg_count = sum(1 for t in tokens if t in negation_words)

        # 特征6-8：情感词比例（使用SentiWordNet快速计算，限制前100个词以加速）
        pos_word_count = 0
        neg_word_count = 0
        sentiment_word_count = 0
        tagged = nltk.pos_tag(tokens[:100])
        
        for word, tag in tagged:
            wn_tag = penn_to_wn(tag)
            if wn_tag is None:
                continue
            synsets = wn.synsets(word, pos=wn_tag)
            if synsets:
                s = swn.senti_synset(synsets[0].name())
                if s.pos_score() > s.neg_score():
                    pos_word_count += 1
                    sentiment_word_count += 1
                elif s.neg_score() > s.pos_score():
                    neg_word_count += 1
                    sentiment_word_count += 1

        sentiment_ratio = sentiment_word_count / len(tokens) if len(tokens) > 0 else 0
        pos_sentiment_ratio = pos_word_count / len(tokens) if len(tokens) > 0 else 0
        neg_sentiment_ratio = neg_word_count / len(tokens) if len(tokens) > 0 else 0

        features.append({
            'word_count': word_count,
            'avg_word_len': avg_word_len,
            'exclamation_count': exclamation_count,
            'upper_ratio': upper_ratio,
            'negation_count': neg_count,
            'sentiment_ratio': sentiment_ratio,
            'pos_sentiment_ratio': pos_sentiment_ratio,
            'neg_sentiment_ratio': neg_sentiment_ratio
        })

    return pd.DataFrame(features)
```

**第2步：提取训练集、验证集、测试集的新特征**

```python

```

**第3步：标准化新特征**

```python
from sklearn.preprocessing import StandardScaler

为什么要标准化？因为不同特征的量纲不同：
- word_count：几百

如果不标准化，数值大的特征会在逻辑回归中占主导地位。

### 模型8：大语言模型（DeepSeek）

#### 模型8a：Few-shot Prompt策略

##### 什么是Few-shot？

给大语言模型几个"示例"，让它学会任务格式，然后对新的输入进行判断。就像考试前给几道例题，让学生理解题型。

##### 我们是怎么做的？

**第1步：配置API**

```python
from openai import OpenAI

client = OpenAI(api_key="your-api-key", base_url="https://api.deepseek.com")
```

**第2步：设计Few-shot Prompt**

```python
few_shot_examples = """You are a sentiment analysis expert. Classify the following movie review as either "positive" or "negative".

Here are some examples:

Review: This movie was absolutely wonderful! The acting was superb and the story was captivating from start to finish.
Answer: positive

Review: Terrible film. The plot made no sense and the acting was wooden. I want my money back.
Answer: negative

Review: A masterpiece of cinema. Every scene was beautifully shot and the performances were outstanding.
Answer: positive

Review: What a waste of time. Boring, predictable, and completely uninspired. The worst movie I've seen this year.
Answer: negative

Review: I really enjoyed this film. It had great characters and an engaging storyline that kept me hooked.
Answer: positive

Review: The movie was okay but nothing special. Some good moments but overall quite forgettable.
Answer: negative

Now classify the following review:
"""
```

这里设计了**6个示例**（3个好评、3个差评），覆盖了不同的表达方式：
- 明确的好评（"absolutely wonderful"）
- 明确的差评（"Terrible film"）
- 模糊的差评（"okay but nothing special"）

**第3步：定义API调用函数**

```python
def call_deepseek(prompt, max_retries=3, max_tokens=10, temperature=0.0):
    """调用DeepSeek API"""
    for attempt in range(max_retries):
        try:
            response = client.chat.completions.create(
                model="deepseek-chat",
                messages=[{"role": "user", "content": prompt}],
                temperature=temperature,      # 0.0确保输出稳定
                max_tokens=max_tokens         # 只需要"positive"或"negative"，10个token足够
            )
            return response.choices[0].message.content.strip()
        except Exception as e:
            print(f"    API调用失败 (attempt {attempt+1}): {e}")
            time.sleep(5)
    return None
```

**第4步：批量测试200条数据**

```python

        pred_str = response.lower()
        if 'positive' in pred_str:
            pred = 1
        elif 'negative' in pred_str:
            pred = 0
        else:
            pred = -1  # 无法判断
        
        results_llm.append({
            'text': text[:200],
            'true_label': label,
            'pred_label': pred,
            'raw_response': response
        })
        
        if pred == label:
            correct += 1
        total += 1
    
    # 限流：每秒最多1次请求
    time.sleep(1.0)

#### 模型8b：结构化输出（框架表示法）

##### 什么是框架表示法？

不仅输出"好评/差评"，还输出更丰富的结构化信息：

```json
{
  "topic": ["acting", "plot"],          // 评论讨论的主题
  "sentiment": "positive",              // 整体情感
  "sentiment_score": 0.85,              // 置信度
  "key_phrases": ["superb acting", "captivating story"],  // 关键短语
  "aspects": [                          // 方面级情感分析
    {"aspect": "acting", "sentiment": "positive", "evidence": "The acting was superb"},
    {"aspect": "plot", "sentiment": "positive", "evidence": "captivating from start to finish"}
  ]
}
```

##### 我们是怎么做的？

**第1步：设计结构化Prompt**

```python
structured_prompt_template = """You are a movie review analyst. Analyze the following movie review and provide a structured output in JSON format.

The output must include:
1. "topic": The main topic(s) of the review (e.g., acting, plot, directing, visual effects, music, etc.)
2. "sentiment": The overall sentiment ("positive" or "negative")
3. "sentiment_score": A confidence score from 0 to 1
4. "key_phrases": List of key phrases that indicate the sentiment
5. "aspects": A list of aspects mentioned with their individual sentiments, each with "aspect", "sentiment", and "evidence"

Example output:
```json

  "sentiment": "positive",
  "sentiment_score": 0.85,
  "key_phrases": ["superb acting", "captivating story"],

```

Provide ONLY the JSON output, no other text.

                'pred_label': pred,
                'parsed_output': parsed
            })
        
        except json.JSONDecodeError:

            pred = 1 if 'positive' in pred_str else 0 if 'negative' in pred_str else -1
            structured_results.append({
                'text': text[:200],
                'true_label': label,

            })
    
    time.sleep(1.0)

## 九、答辩可能被问到的问题

### Q1：为什么SentiWordNet准确率这么低？

三个原因：(1)只看词的情感极性，忽略上下文和语序；(2)无法处理否定（如"not bad"中的bad被计为负向）和反讽；(3)取第一个同义词集可能不准确。误分类中假正向远多于假负向，说明它倾向于将模糊样本判为正向。

### Q2：特征选择K=2000为什么是较好的平衡点？

K=200时特征太少，丢失了重要情感信息，准确率下降6个百分点；K=2000时准确率88.68%，仅比全特征低0.86个百分点，但特征维度从50000降至2000（降低96%），计算成本大幅减少。

### Q3：新特征为什么没有显著提升？

8个手工特征单独使用有62.54%的准确率，说明包含情感信息。但与TF-IDF（50000维）拼接后，8维新特征的信号被稀释。改进方向：对特征进行归一化或使用特征交叉。

### Q4：Few-shot Prompt为什么效果这么好？

大语言模型通过预训练学习了丰富的语言知识，6个示例足以引导模型理解任务格式。模型能理解否定、反讽等复杂语义，这是传统方法难以做到的。

### Q5：结构化输出的框架表示有什么意义？

框架表示不仅给出整体情感判断，还提供主题提取、方面级情感分析和关键短语，实现了细粒度分析。例如，一条评论可能对演技正面但对剧情负面，这种信息对实际应用更有价值。

### Q6：PMI分析发现了什么有趣的规律？

PMI值最高的词几乎都是专有名词——正向PMI词如dassin（经典导演）、gundam（知名系列），负向PMI词如manos（著名烂片）、uwe boll（以烂片闻名的导演）。这说明专有名词具有强烈的情感指向性。

## 十、总结：你应该记住的核心要点

1. **任务本质**：判断电影评论是好评还是差评——一个文本二分类问题
2. **三种技术路线**：规则方法（查词典）→ 机器学习（从数据学习）→ 大语言模型（利用预训练知识）
3. **性能递进**：62% → 89.5% → 95.4%，数据驱动远胜人工规则，大模型又远胜传统方法
4. **特征工程的价值**：TF-IDF比词频好，2000个关键特征就够用，否定词和负向情感词是最重要的手工特征
5. **PMI的洞察**：专有名词（电影名、导演名）具有强烈的情感指向性
6. **SentiWordNet的定位**：作为规则方法性能有限，但情感词信息可作为特征工程参考
7. **大语言模型的优势**：无需特征工程，Few-shot即可达到95%+，结构化输出能实现细粒度分析
